## Seeded from upstream (2026-06-10)

### Stakes & audience
- **Stakes:** internal team-building tool (hobby/indie tier). Not commercial. Single self-hosted Docker Compose deployment.
- **Audience:** internal teams (engineering, product, design, data, management). 2–16 players per session, organized by a team lead.
- **Session length:** 45–90 min including setup, rounds, and debrief.

### Form-factor & inputs
- **Surfaces:** desktop browser only in V1. Chrome 90+, Firefox 88+, Edge 90+ primary. Safari second-class.
- **Mobile:** responsive acceptable, but not optimized in V1.
- **Inputs:** mouse + keyboard for all UI; microphone required for voice; browser WebRTC (LiveKit). No controller, no touch, no VR.
- **No accounts, no downloads, no external voice tools.**

### Roles (four)
1. **Defuser** — sees the 3D bomb only. Cannot see manual. Mouse-clicks modules.
2. **Expert(s)** — sees digital manual only. Cannot see bomb. Voice instructs Defuser. Asymmetric Expert Roles (split chapters) optional.
3. **Spectator** — appears when >2 players. Read-only view of bomb + Expert's current manual page side-by-side. Optional lifeline token mechanic. Listen-only voice.
4. **Facilitator** — non-playing. Dashboard for session config, team assignment, round pacing, retry, spectator chat visibility toggle.

### Voice topology
- **Bomb Room channel:** Defuser + Experts + Facilitator (bidirectional).
- **Spectator Lounge channel:** Spectators (listen-only to Bomb Room; cannot broadcast in).
- **Mute toggle per player.** Visual speaker indicator.
- **Facilitator PTT bridge** between channels — most likely failure point per project-context.

### Surfaces (every screen/view named in sources)
- Lobby (join code, team display, ready state)
- Preparation phase (all players browse manual; Defuser sees module types only, no values)
- Bomb view — Defuser-facing 3D R3F scene with timer + strike counter + mute
- Manual viewer — Expert-facing; chapter select + page scroll + tables
- Spectator lounge view — bomb + Expert's current manual page side-by-side
- Facilitator dashboard — session config, team assignment, round control, retry, chat-visibility toggle
- Voice channel UI — mute, speaker indicator
- Between-round scoreboard preview
- End-of-session final scoreboard
- Lifeline overlay (optional, banner)
- Module sandbox `/dev/sandbox` (dev-only, not player-facing)

### HUD information hierarchy (Defuser bomb view)
- Always visible: countdown timer (server-authoritative), strike counter (0/1/2), per-module green-LED solve state.
- Strike escalation: timer speed +25% per strike (compounding; configurable 0–50%).
- Speaker indicator overlay (who is talking in Bomb Room).
- Mute toggle.
- No individual strike attribution — team-wide.

### Modules (11, tiered)
- **Easy:** Wires, The Button, Passwords.
- **Medium:** Keypads, Who's on First, Wire Sequences, Mazes.
- **Hard:** Complicated Wires, Simon Says, Memory, Morse Code.
- **V2 / out of scope:** Needy modules (Venting Gas, Capacitor Discharge, Knobs).

### Aesthetic / brand
- **Stylized retro-industrial 1960s spy-thriller prop.** Not realistic military hardware. Cartoony but not ridiculous.
- **Chunky, bold colors, thick outlines.** KTANE module-language reference.
- **Diegesis:** bomb is the protagonist — modules feel like physical hardware. Timer, strike counter, LEDs are *in-world* on the bomb chassis. Voice/mute/speaker indicators are *non-diegetic* overlays.
- **No music during rounds — voice is the soundtrack.** Minimal SFX: tick, solve chime, strike, explosion, defuse fanfare.
- **Naming:** Bomb Squad, Defuser, Expert, Facilitator, Bomb Room, Spectator Lounge.

### Accessibility floor
- **Colorblind accommodation required before V1 release** — modules Wires, The Button, Simon Says, Complicated Wires use color as primary descriptor. Patterns/labels mandatory.
- No captioning called out (voice channel is inherently spoken; future consideration).
- No screen-reader target in V1 (visual game).
- Motor: all interactions are mouse clicks + voice; no rapid-input or dexterity demands.

### Design constraints
- 60fps gate on bomb view (treat any violation as a bug).
- Module visuals must be describable in plain language under 60s of time pressure → unambiguous, high-contrast component design required.
- Drop-in/drop-out unsupported mid-session → disconnect triggers auto-pause modal.
- English only V1.
- Spectator-coaching risk → facilitator toggles spectator chat visibility.
